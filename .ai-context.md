## Quick Summary

**Internal Trello** — Kanban Board cho mạng nội bộ, tương tự Trello.
- **Backend**: Node.js + Express + `better-sqlite3` (sync SQLite, KHÔNG async)
- **Frontend**: Vanilla HTML + CSS + JS (SPA, không framework, không build step)
- **Auth**: Session-based (`express-session`, cookie 24h, in-memory store)
- **Port mặc định**: 3000 (`.env`)
- **Chạy**: `cd internal-trello && node server.js`

## Architecture

```
Browser (SPA)
├── index.html          ← 1 file duy nhất (tất cả screens + modals)
├── css/style.css       ← Design system (biến CSS, dark theme, glassmorphism)
└── js/
    ├── api.js          ← API client (mọi call fetch đều qua đây)
    ├── auth.js         ← Login/Register UI
    ├── board.js        ← Board list UI
    ├── kanban.js       ← Kanban UI (columns, cards, drag-drop, modals)
    └── app.js          ← Controller (routing, toast, admin, members, labels)
          │ HTTP
          ▼
server.js               ← Express entry point
├── /src/middleware/auth.js      ← requireAuth / requireAdmin
├── /src/db/database.js          ← Schema, seed admin, avatar colors
└── /src/routes/
    ├── auth.routes.js           ← /api/auth/*
    ├── board.routes.js          ← /api/boards/* + members endpoints
    ├── column.routes.js         ← /api/columns/*
    ├── card.routes.js           ← /api/cards/* + comments
    └── label.routes.js          ← /api/boards/:id/labels + /api/cards/:id/labels
          │ SQL (sync)
          ▼
data/kanban.db          ← SQLite file (tự tạo khi chạy lần đầu)
```

**Global JS Objects** (thứ tự load trong index.html):
`API` → `Auth` → `Board` → `Kanban` → `App`

## Database Schema (đầy đủ)

| Table | Columns quan trọng |
|---|---|
| `users` | `id, username, password(bcrypt), display_name, role(admin/user), avatar_color, is_active, created_at, last_login` |
| `boards` | `id, title, description, background(CSS gradient string), owner_id, created_at, updated_at` |
| `board_members` | `board_id, user_id, role(admin/member), joined_at` |
| `columns` | `id, board_id, title, position(int 0-based), created_at` |
| `cards` | `id, column_id, title, description, position(int), priority(low/medium/high/urgent), due_date, created_by, assigned_to(→users.id nullable), created_at, updated_at` |
| `labels` | `id, board_id, name, color(hex string)` |
| `card_labels` | `card_id, label_id` (junction table nhiều-nhiều) |
| `comments` | `id, card_id, user_id, content, created_at` |

> **`better-sqlite3` là SYNC (không phải async)**. Dùng `db.get()`, `db.all()`, `db.run()` — KHÔNG dùng `.then()` hay `await`.

## API Endpoints (đầy đủ v2.0)

### Auth — `/api/auth/`
| Method | Path | Auth | Body |
|---|---|---|---|
| POST | `/login` | - | `{username, password}` |
| POST | `/register` | - | `{username, password, displayName}` |
| POST | `/logout` | session | - |
| GET | `/me` | session | - |
| GET | `/users-list` | session | → `[{id, username, displayName, avatarColor}]` dùng cho invite dropdown |
| GET | `/admin/users` | admin | → DS tất cả users |
| POST | `/admin/create-user` | admin | `{username, password, displayName, role}` |
| PUT | `/admin/reset-password/:userId` | admin | `{newPassword}` |
| PUT | `/admin/toggle-active/:userId` | admin | - |

### Boards — `/api/boards/`
| Method | Path | Auth | Ghi chú |
|---|---|---|---|
| GET | `/` | session | DS boards mà user là owner hoặc member |
| POST | `/` | session | Tạo board + tự add owner vào board_members (role=admin) + tạo 3 cột mặc định |
| GET | `/:id` | member | **Response đầy đủ**: board + columns + cards (với labels + assignee) + members |
| PUT | `/:id` | session | Sửa board |
| DELETE | `/:id` | owner/admin | Xóa cascade (card_labels → comments → cards → columns → labels → board_members → board) |
| GET | `/:id/members` | member | DS members |
| POST | `/:id/members` | board admin | `{userId}` — thêm member |
| DELETE | `/:id/members/:userId` | board admin/self | Xóa member (không xóa được owner) |

### Columns — `/api/columns/`
| Method | Path | Body |
|---|---|---|
| POST | `/` | `{boardId, title}` |
| PUT | `/:id` | `{title}` |
| DELETE | `/:id` | - (xóa cascade cards + card_labels + comments) |
| PUT | `/reorder/batch` | `{columns: [{id, position}]}` |

### Cards — `/api/cards/`
| Method | Path | Body |
|---|---|---|
| POST | `/` | `{columnId, title, description?, priority?}` |
| PUT | `/:id` | `{title, description, priority, dueDate, assignedTo}` |
| DELETE | `/:id` | - |
| PUT | `/:id/move` | `{columnId, position}` |
| GET | `/:id/comments` | - |
| POST | `/:id/comments` | `{content}` |

### Labels — `/api/` (mount tại root `/api`)
| Method | Path | Body |
|---|---|---|
| GET | `/boards/:boardId/labels` | - |
| POST | `/boards/:boardId/labels` | `{name, color}` |
| DELETE | `/labels/:id` | - (xóa cả card_labels liên quan) |
| GET | `/cards/:cardId/labels` | - |
| POST | `/cards/:cardId/labels` | `{labelId}` |
| DELETE | `/cards/:cardId/labels/:labelId` | - |

## Bug Fix Cheatsheet — "Sửa cái này đừng quên cái kia"

| Triệu chứng | File cần kiểm tra | Điểm mấu chốt |
|---|---|---|
| Card không hiện label trên board | `board.routes.js` GET `/:id` | Query JOIN labels tại dòng ~115 |
| Label picker trống khi mở card | `kanban.js` `openCardModal()` | `this.boardLabels` phải được load trong `loadBoard()` |
| Assignee dropdown trống | `kanban.js` `openCardModal()` | `this.boardMembers` phải được load từ `currentBoard.members` |
| Không lưu được assignee | `kanban.js` `saveCard()` | `assignedTo: parseInt(assignedTo)` — phải parseInt |
| Member modal trống | `app.js` `openMembersModal()` | `Kanban.currentBoard.id` phải có giá trị |
| 401 khi load /api/auth/me | Bình thường khi chưa login | Không phải bug |
| Xóa board không xoá hết data | `board.routes.js` DELETE | Thứ tự: card_labels → comments → cards → columns → labels → board_members → boards |
| Session mất sau restart server | Cố ý: in-memory store | Để persist: dùng `connect-sqlite3` (Phase 3) |
| `db.get is not a function` | Kiểm tra import | `const { database } = require('./src/db/database')` rồi `app.locals.db = database` |
| State không đồng bộ sau thao tác | Kiểm tra có `loadBoard()` không | Sau mọi thay đổi quan trọng phải `Kanban.loadBoard(id)` |

## State Management (Frontend)

```
Kanban.currentBoard   // Board đang mở (full data: columns, cards, members)
Kanban.boardMembers   // Cache members (dùng cho assignee dropdown)
Kanban.boardLabels    // Cache labels (dùng cho label picker)
Kanban.activeCardLabels // Label IDs đang được chọn trong card modal (mảng số)
Kanban.editingCardId  // Card đang mở modal chi tiết
```

> ⚠️ Sau khi mời/xóa member, phải cập nhật `Kanban.boardMembers` (xem `app.js` `inviteMember()`)
> ⚠️ Sau khi tạo/xóa label, phải cập nhật `Kanban.boardLabels` (xem `app.js` `openLabelsModal()`)

## Conventions (bắt buộc theo để tránh bug)

1. **`parseInt()` mọi ID** từ params/body trước khi dùng trong SQL
2. **Không inline onclick** trong HTML động — dùng `addEventListener` sau khi render
3. **Escape HTML** khi render user input: `Board.escapeHtml(str)`
4. **Reload board** sau mọi thao tác thay đổi data: `Kanban.loadBoard(Kanban.currentBoard.id)`
5. **Validate bắt buộc** trước mọi INSERT/UPDATE: kiểm tra field required
6. **Xóa cascade** theo đúng thứ tự: xóa child trước, parent sau
7. **CSS class** theo pattern: component-element-modifier (ví dụ: `card-label-pill`, `member-role-badge`)

### 🔑 Checkpoint 14: Auth Middleware và Tên Biến Hệ Thống
- User ID trong các Controller không được lấy bằng `req.user.id` mà bắt buộc là **`req.session.userId`**.
- Middle ware của Application Trello này không sử dụng `requireRole('admin')` mà chỉ export thẳng hàm `requireAdmin` và `requireAuth`.
- Nếu viết Route mới cho Admin sử dụng Auth dạng: `router.post('/', requireAuth, requireAdmin, (req, res) => {...})`.

### 🔑 Checkpoint 15: Socket.IO Realtime & Multi-Assignee
**File**: `server.js` — Socket.IO được tích hợp qua `http.createServer(app)` + `new Server(server)`.
- Server lắng nghe `join-board` / `leave-board` để quản lý Room theo Board ID.
- Để phát tín hiệu Realtime từ bất kỳ Route nào: `req.app.get('io').to('board-' + boardId).emit('board-updated', { action: 'tên-hành-động' })`.
- Frontend kết nối Socket 1 lần duy nhất trong `Kanban.initSocket()`, lắng nghe `board-updated` → gọi `silentReload()`.
- `silentReload()` chỉ reload DATA (không toast, không đổi view, không ảnh hưởng drag-drop đang diễn ra).

**Multi-Assignee Architecture:**
- Bảng `card_assignees` (card_id, user_id) — giống pattern `card_labels`.
- API nhận `assigneeIds: [1, 2, 3]` (mảng) thay vì `assignedTo: 1` (số đơn).
- Frontend dùng `this.activeCardAssignees` (mảng IDs) — giống `this.activeCardLabels`.
- Render tối đa 3 Avatar xếp chồng + badge "+N" ngoài bảng.

### 🔑 Checkpoint 16: Hạn chế của sql.js & Lỗi Transaction "No transaction is active"
- **Không dùng thủ công `BEGIN TRANSACTION / COMMIT` trong Backend**: Do cơ chế Wrapper lưu Data tự động (`db.export() -> fs.writeFileSync()`) sau mỗi hàm `db.run`, việc sử dụng mở/đóng thủ công giao dịch sẽ gây xung đột, SQLite sẽ tự `ROLLBACK` và ném lỗi treo Backend Server (ví dụ đối với Batch Process trong Thùng rác).
- Cứ dùng `db.run` từng dòng 1 bình thường, `sql.js` sẽ lo chuyện auto-commit tự động với tốc độ tính bằng phần nghìn giây. Vì là Web nội bộ, đảm bảo Auto-commit an toàn và hiệu quả hơn rất nhiều so với tự config transaction!
